Binary Search Game

In this project, we made a text based game in JS based on Binary Search. We played a game in class to understand the concept and drew out the algorithm for our game with a flow chart. My group chose to make a game where the program would guess the US state that the user thought of. The program would ask the user questions about geography and position of the state relative to other states or geographic features of the US. We collaborated on this project by using Pair Programming. We learned about Boolean Expressions and Selection Conditionals to help with the searching process. This project also made us learn a lot about Identifying and Correcting Errors.

Binary Search is the concept of dividing the amount of items in a list by two until there is only one possibility for what the item could be. What that means to my group's game and the user interface is that the program asks the user yes or no questions in relation to the state they chose, utilizing readyesorno functions. The user would write yes or no depending on how the question related to their state. Boolean Expressions are used to make the program function differently depending on if the user wrote yes or no. You set a variable to something related to the question. In simple terms, you use if statements and else statements. If statements are used when the user answers yes, the boolean expression is if the variable = true, then do this. Else statements have no boolean expression, because you can assume the variable = false, so so you tell the program to do something differently compared to if the variable is true. Selective Conditionals are functions we used when we needed to divide by 3 or 4. For those cases we would use a readchoice function to choose between different terms assigned to a number. The user would write the number that stood for the term that was correct to the state they chose. Pair Programming is a collaboration method used for writing code. The model for three people in a group is one person actually writing the code, one person supervising the code writer to check for errors, and one person mapping out the path of the code with a flow chart. Our group used Identifying and Correcting Errors when we had written the entire code, and we were testing to make sure the game worked, by testing inputs to see if the game guessed the correct state. When we noticed an error we went to the part of the code where the error occurred and tried to notice if there was something wrong with the code. When we found the error and fixed it, then we checked the game in that spot again to make sure it was fixed.

APCSP Portfolio

This project is a portfolio used to showcase our work done in this class. It's also to explain what this class is about. We learned a lot about the Design Process of making a website, and user interface. We learned about Hosting and the Internet with how our website needs a server to be seen by anyone. We learned about the languages HTML and CSS and how they are used in the making of a webpage. We also learned about Researching and Using Libraries to enhance our webpages. This project really challenged my ability to use all three languages to make my portfolio exactly as I want it to be.

Design Process was used when we wanted to plan out how we would implement certain elements into our webpage, and setting SMART goals for the work that we would do on work days. Hosting and the Internet is about hosting your website on a hosting server so it can be viewed on the internet. The hosting server we used is Netlify. We needed to host our source code to a code repository to document changes made to the code and save it in case something goes wrong. The code repository we used in Github.HTML is the language used to get the basic layout of the webpage, making elements and implementing other media where they are meant to go, and putting links. CSS is the language used to make the page look exactly how you want it to. You can take the elements defined in HTML and change things about them, like color, size, adding borders, etc. JS is the language used to make everything work, like menus, downloading libraries, importing media to be put into links, etc. Researching and Using Libraries was not used by me, because I ran out of time. I tried to researching libraries, but I didn't get very far. Libraries are pre coded sections of code that are used to implement difficult things into webpages, like animations, transitions, backgrounds, etc.
